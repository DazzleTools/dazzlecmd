## What this is

v0.7.23 is Phase 1 of the Option C wtf-windows-adoption work. It ships the library surface that wtf-windows (and future aggregators) will adopt:

- `MetaCommandRegistry` — per-engine meta-command registry (new primary API)
- `default_meta_commands` — stock `list`/`info`/`kit`/`version`/`tree`/`setup`
- `cli_helpers` — shared argparse scaffolding
- Per-aggregator config_dir + override_root isolation
- Engine integration with registry path as primary, callback path as escape hatch

Phase 2 (next commit, wtf-windows repo) will adopt this library surface.

- [ ] **Steps** (cross-shell, uses Python only):

          version_info=('1.0', '1.0.0_test_1'),
          config_dir=td,

### HV.3 -- Override handler keeps stock parser

Validates: `override("info", handler=...)` replaces handler without duplicating parser definition.

- [ ] **Steps**:
  ```
  python -c "
  import tempfile
  from dazzlecmd_lib import AggregatorEngine
  with tempfile.TemporaryDirectory() as td:
      engine = AggregatorEngine(name='t', command='t', config_dir=td, version_info=('1','1'))
      original_parser, original_handler = engine.meta_registry.resolve('info')

      def custom_handler(args, eng, projects, kits, project_root):
          print(f'CUSTOM INFO for {args.tool}')
          return 0

      engine.meta_registry.override('info', handler=custom_handler)
      new_parser, new_handler = engine.meta_registry.resolve('info')
      print('Parser unchanged:', new_parser is original_parser)
      print('Handler replaced:', new_handler is custom_handler)
  "
  ```

- [ ] **Expected**:
  - `Parser unchanged: True`
  - `Handler replaced: True`

## Detailed verification

### Section 1: Registry CRUD operations

#### 1.1 register() raises on duplicate

- [ ] ```python
  from dazzlecmd_lib import MetaCommandRegistry, MetaCommandAlreadyRegisteredError
  r = MetaCommandRegistry()
  r.register('x', lambda s: None, lambda *a: 0)
  try:
      r.register('x', lambda s: None, lambda *a: 0)
      print('FAIL: should have raised')
  except MetaCommandAlreadyRegisteredError:
      print('PASS: raised as expected')
  ```

#### 1.2 override() raises on unknown name

- [ ] ```python
  from dazzlecmd_lib import MetaCommandRegistry, MetaCommandNotRegisteredError
  r = MetaCommandRegistry()
  try:
      r.override('never', handler=lambda *a: 0)
      print('FAIL')
  except MetaCommandNotRegisteredError:
      print('PASS')
  ```

#### 1.3 Registry locks during dispatch

- [ ] ```python
  from dazzlecmd_lib import MetaCommandRegistry, RegistryLockedError
  r = MetaCommandRegistry()
  r.register('x', lambda s: None, lambda *a: 0)
  r.lock()
  try:
      r.register('y', lambda s: None, lambda *a: 0)
      print('FAIL')
  except RegistryLockedError:
      print('PASS')
  ```

### Section 2: Default meta-commands

#### 2.1 `list` subparser accepts all filters

- [ ] ```python
  import argparse
  from dazzlecmd_lib import default_meta_commands as dmc
  parser = argparse.ArgumentParser()
  subs = parser.add_subparsers(dest='command')
  dmc.list_parser_factory(subs)
  args = parser.parse_args(['list', '-n', 'core', '-k', 'mykit', '-t', 'audit', '-p', 'linux'])
  assert args.namespace == 'core'
  assert args.kit == 'mykit'
  assert args.tag == 'audit'
  assert args.platform == 'linux'
  print('PASS')
  ```

#### 2.2 `info` with FQCN disambiguates collisions

- [ ] ```python
  from dazzlecmd_lib import default_meta_commands as dmc
  from types import SimpleNamespace
  projects = [
      {'name': 'x', '_fqcn': 'kit1:x'},
      {'name': 'x', '_fqcn': 'kit2:x'},
  ]
  # Short-name lookup ambiguous
  rc = dmc.render_info(SimpleNamespace(tool='x'), projects)
  assert rc == 1
  # FQCN unique
  rc = dmc.render_info(SimpleNamespace(tool='kit2:x'), projects)
  assert rc == 0
  print('PASS')
  ```

### Section 4: Per-aggregator overrides

#### 4.1 Override root routes to engine's config_dir

- [ ] ```python
  import tempfile, os, json
  from pathlib import Path
  from dazzlecmd_lib import AggregatorEngine, user_overrides

  with tempfile.TemporaryDirectory() as td:
      # Create an override file at the per-aggregator location
      overrides = Path(td) / 'overrides' / 'setup'
      overrides.mkdir(parents=True)
      (overrides / 'kit__tool.json').write_text(
          json.dumps({'_schema_version': '1', 'command': 'echo AGG_OVERRIDE'})
      )
      engine = AggregatorEngine(name='t', command='t', config_dir=td)
      result = user_overrides.load_override('setup', 'kit:tool')
      assert result is not None
      assert result['command'] == 'echo AGG_OVERRIDE'
      print('PASS')
  ```

#### 4.2 DAZZLECMD_OVERRIDES_DIR env var still takes precedence

- [ ] **cmd.exe**:
  ```cmd

  set DAZZLECMD_OVERRIDES_DIR=

- [ ] **PowerShell**:
