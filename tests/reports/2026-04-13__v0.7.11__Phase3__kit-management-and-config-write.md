## Edge Cases Discovered

### 4.6 -- `dz tree --show-disabled` is a no-op (BUG)

**Root cause**: `dz tree` builds its kit-to-tools mapping from `engine.projects`, which is the already-filtered discovery result. When a kit is disabled, its projects are never discovered by the engine, so `by_kit` (line ~1350 in cli.py) never contains a key for the disabled kit. The `--show-disabled` flag (line 1395) only controls whether disabled kit names are *filtered out of* `kit_names`, but if the kit was never in `by_kit` to begin with, there is nothing to show.

**Reproduction**:
```
dz kit disable dazzletools
dz tree                    # no dazzletools (correct)
dz tree --show-disabled    # still no dazzletools (BUG -- should show it)
```

**Fix direction**: The tree command would need a secondary discovery pass with all kits enabled (or access to the full kit registry independent of the active-kit filter) to populate disabled kit entries. Alternatively, `engine.kits` could always include all registered kits regardless of enable/disable state, with a flag indicating their status.

**Affected code**: `src/dazzlecmd/cli.py` lines 1348-1399 (the `_handle_tree` function).
